Use REM

Motivos para o uso da unidade "rem" do CSS para Web App's

Você, provavelmente, deve conhecer e usar a unidade do CSS EM em seus projetos.

Ela é uma unidade muito útil para alterar dimensões da interface de um projeto - quando é bem estruturado - alterando-se a unidade em pixels em apenas alguns locais.

Porém, usar em tem suas complicações. Observe a estrutura e estilos abaixo:

<body>
    <p>
        <a></a>
    </p>
</body>

body {
    font-size: 10px;
}
p {
    font-size: 2em;
}
a {
    font-size: 4em;
}

O resultado disso sera que p terá font-size = 20px e a terá font-size = 80px. Isto ocorre porquê a referência da unidade em é o elemento html pai.

A unidade REM, sendo que o R significa root, se referencia a tag raiz <html>.

Desta forma, todos os elementos que são definidos com rem serão alterados quando a unidade em px da tag html for alterada. Observe abaixo:

<html>
    <body>
        <p>
            <a></a>
        </p>
    <html>
</body>

html {
    font-size: 10px;
}
p {
    font-size: 2rem;
}
a {
    font-size: 4rem;
}

Como resultado, p terá font-size = 20px e a terá font-size = 40px, pois a referência agora é a tag html.

Estes tipos de unidade não são utilizadas apenas para alterar tamanhos de fonte. Pode ser usada para definir larguras, margens, padding, etc. Pois, tanto em quando rem são medidas percentuais.

A grande vantagem de construir um Web App usando todas as medidas em rem é que você terá muito menos trabalho para redefinir estas medidas para os vários tamanhos de telas e as variações de sistemas operacionais para os quais seu aplicativo será distribuído.

Basta alterar o font-size da tag html.
